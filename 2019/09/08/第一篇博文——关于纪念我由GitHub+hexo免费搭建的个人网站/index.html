第一篇博文——关于纪念我由GitHub+hexo免费搭建的个人网站

2019年9月8日

​ 事实上早在两个月前，在看过B站up主codesheep的视频讲解后，我就搭建了人生的第一个个人网站，当时由于第一次尝试，电脑中没有麻烦的子文件骚扰，搭建过程也十分的顺利（当时完全按照up主codesheep的指点，类比其在mac上的root用户进行操作，我在win10上用cmd的操作过程也颇为顺利，但是搭建完之后，由于缺乏时间运营，hexo版本最终过期，_ config.yml文件也最终损坏，且由于当时安装的各类编译器将_config.yml文件散乱保存，直接导致这一次的操作遇阻。话不多说，接下来就是我漫长的博客网站重搭建之路。。。

一、hexo -s失效，git无法将做好的博文推出，开启重装之路。

​ 说起来，现在这段代码对我来说简直是噩梦。我想你会理解我，假如你尝试十几遍的相关软件卸载重装，（其实重装并不可怕，可怕的校园网的龟速*外网下载的龟速 *老式商务笔记本带来的三次降维打击。

​ 事实上这一段为什么出错，我也不是特别清楚，但经过十几遍重装的排查，基本可以确定不是Git环境或者是node.js的问题（一开始我以为是Git和node.js版本不对口，白白重装了好久的软件），我认为很大程度上是出自原文件删除不干净造成的（可以这么认为，跟原老婆的关系还没完全断开，就急着跟新老婆结婚，新老婆当然不愿意）

​ 当然前面这些也是我的一个猜测，至于是不是，来日如果有排查清楚，会做一个专门的后续，这里就不多赘述。

二、多方支援，寻觅方法（找对方法很重要）

​ 在重试了五六遍codesheep的方法之后，我确定了只删除原blog文件以及卸载重装node.js以及Git无法成功，又查了不下一百篇博文，各种修补漏洞和补丁的办法也最终破产。到了最后，我决定尝试CSDN上一位博主zjufangzh的方法。这里是搭建博客的方法，个人觉得非常详尽有条理，大家如果愿意搭建网站的话可以看一下

https://blog.csdn.net/sinat_37781304/article/details/82729029

​ 注意！注意！注意！如果你与我一样是尝试过n遍依然遇到那该死的Usage: hexo时，千万别跟我一样忙着重装卸载，一定要把之前安装留下的蛛丝马迹清除掉，另外在修改_config.yml文件时要分外小心，不能出错。一定要严谨，一下是我修改之后的样式，仅供参考（大家可以复制但千万要把名字改掉）

注意冒号后一定要有一个空格，下方最好多出一行。

三、发博文的方法

​ 如果你成功搭建完个人网站，并且成功进行了域名绑定，那么接下来就好好开一瓶可乐庆祝一下吧。

​ 回归正题，我想推荐几个用来写博文的软件，接下来的几个软件是我自己习惯的几个软件（附下载地址），大家参考就行了。

​ 1、Typora https://typora.io/

​ 只是我个人用起来最顺心的一个网页文档编辑器，功能强大，编译简单。

​ 2、word （问参考地址的去面壁）

​ 文档编辑器也是很好的一个撰写博文方法，就是后期需要将文件改为.md，大家看着用吧。

​ 3、记事本（这个和word类型一样，好处在于每天window机子都有）

​ 4、notepad++ https://notepad-plus-plus.org/

​ 这个软件的功能很强大，可以作为很多语言的编译器，但用来写博文可能稍微难受一点。

​

​ 博文的最后，推荐一下强大的华为爸爸，不，华为云爸爸，在这次的下载外网软件时，我深刻意识到一个镜像软件收集网站的重要性（附上华为云镜像软件群的地址：https://mirrors.huaweicloud.com/）

​ 另外，衷心感谢CSDN博主zjufangzh带来的莫大帮助，还有B站up主codesheep带来的视频讲解。
